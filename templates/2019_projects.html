Initial Concept Design

After multiple iterations of the design it was decided to use a two powered wheel one castor approach instead of the 4 wheel design. The 4 wheel design presented challenges like increased turning radius and almost 2x the cost for motors. In the final design steering is achieved by differentials.

Android App Control Model

For manually controlling the robot an Android app was written which was capable of communicating with the robot via Bluetooth. This signal was passed onto the Arduino which governs the speed of each motor. A simple PD control model was sufficient for this task.

'Battleship' Game on Embedded Controllers

December 2019

Using concepts such as state machines, debouncing and EEPROM permanance the game 'Battleship' was designed on a ATMEGA 128 MCU. Several buttons and a mini LED array were used to depict the positions of the ships ease navigation. Additionally multiplayer mode was also enabled by interconnected RX - TX lines of two MCUs for true 'Battleship' experience.

Video Demo of Battleship

Demo shows how a map can be uploaded to the ATMEGA MCU from the PC via a UART interface. The map is transmitted one byte at a time to the MCU via the TX/RX pins. Hardward interrupts are used to properly read the incoming signal. Once all 64 bytes (8x8 map) have been read the MCU saves this data to it's internal EEPROM. The game can then be played as usual using the buttons. Stats and other info are displayed on the 16x2 LCD panel.
